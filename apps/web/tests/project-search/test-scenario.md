## 사전조건

- 검색 가능한 프로젝트가 1개 이상 존재
- localhost: Turnstile siteKey 빈 문자열 → 위젯 미렌더 (봇 검증 자동 통과)
- preview/prod 자동화: always-pass 테스트 키 적용 → 골든패스가 위젯 통과까지 자연 검증

## Golden path - 부분 키워드 검색 → 리스트에서 선택 → 인증 → 대시보드

- 랜딩 페이지에서 내 프로젝트 찾기 버튼을 클릭한다.
- 검색 모달이 출력된다.
- 부분 키워드 입력
  - 결과 리스트에 해당 프로젝트가 출력된다.
- 결과리스트에서 프로젝트를 클릭한다.
  - 식별번호 입력 페이지로 이동한다.
- 올바른 식별번호를 입력하고 제출한다.
- 프로젝트 대시보드로 리디렉트되고 프로젝트 이름이 페이지에 출력된다.

## Golden path - 정확한 이름 검색 → 검색 버튼 제출 → 인증 → 대시보드

- 랜딩 페이지에서 내 프로젝트 찾기 버튼을 클릭한다.
- 검색 모달이 출력된다.
- 존재하는 프로젝트의 정확한 이름을 입력한다.
- 검색 버튼을 클릭한다.
- 인증페이지로 리디렉션
- 식별번호 입력한다.
- 프로젝트 대시보드로 리디렉트되고 프로젝트 이름이 페이지에 출력된다.

## cancel - 이탈 시나리오

### 검색모달 닫기

- 닫기 버튼(x)을 클릭하면 모달이 닫히고 랜딩페이지에 머무른다.
- ESC 키를 입력하면 모달이 닫힌다.
- 모달 외부 화면을 클릭하면 모달이 닫힌다.

### 식별번호 입력 페이지 이탈

- 뒤로가기 시 랜딩페이지로 이동한다.
- 다시 프로젝트 식별번호 입력 페이지 진입시 입력값이 비어있다.

## bot check - Cloudflare Turnstile 차단 (fixme, preview 한정)

> 위젯 통과(always-pass)는 사전조건 그대로 골든패스에서 자연 검증됨 — 본 섹션은 차단(negative) 케이스만 다룬다.
> always-fail 키가 적용된 별도 환경 또는 런타임 토글이 필요하므로 현재는 test.fixme 로 placeholder.

### (fixme) 위젯 차단 — always-fail 환경

- 식별번호 입력 페이지에서 Turnstile 위젯이 렌더된다.
- 위젯이 fail 상태가 된다.
- 올바른 식별번호를 입력해도 제출이 차단된다 (버튼 disabled 또는 서버 거부).
- 봇 차단 에러 UI 가 노출되고 재시도가 가능하다.
