## 프로젝트 생성 시나리오

### Test Purpose

사용자가 프로젝트 생성을 완료하고 생성한 프로젝트로 접근이 가능한지 검증한다.

### Pre-condition

- 랜딩페이지가 렌더링 되어야한다.

### Scenario

1. 랜딩페이지에 접근한다.
2. 안내 모달이 출력된다면 종료해야한다.
3. 무료로 시작하기 버튼을 클릭한다.
4. (Step 1) 프로젝트 제목을 입력한다.
5. (Step 1) 이용약관 체크박스를 체크한다.
6. (Step 1) 프로젝트 생성 시작 버튼을 클릭한다.
7. (Step 2) 식별번호/식별번호 확인을 입력한다.
8. (Step 2) 프로젝트 생성하기를 클릭한다.
9. (Step 3) 프로젝트 정보가 출력되며 생성하기를 클릭한다.
10. (Step 4) 프로젝트 생성이 완료되며 링크복사 버튼 클릭시 프로젝트 URL이 복사된다.
11. (Step 4) 시작하기 버튼을 클릭하면 대시보드로 이동한다.

- Feature: 프로젝트 생성
- Scenario: 사용자가 랜딩에서 프로젝트 생성을 완료하면 생성한 프로젝트 대시보드로 진입할 수 있다
- Given
  - 비인증 상태로 랜딩페이지가 렌더링되어 있다.
- When
  - 랜딩페이지에 접근한다.
  - 안내 모달이 떠 있으면 닫는다.
  - `무료로 시작하기` 버튼을 클릭한다.
  - (Step 1) 고유한 프로젝트 제목을 입력하고, 이용약관 동의·만 14세 이상 확인 체크박스를 체크한 뒤 `프로젝트 생성 시작` 버튼을 클릭한다.
  - (Step 2) 식별번호와 식별번호 확인을 입력하고 `프로젝트 생성하기` 버튼을 클릭한다.
  - (Step 3) 출력된 프로젝트 정보를 확인하고 `생성하기` 버튼을 클릭한다.
  - (Step 4) `링크 복사` 버튼을 클릭한다.
  - (Step 4) `시작하기` 버튼을 클릭한다.
- Then
  - (Step 4) 프로젝트 생성 완료 화면이 표시된다.
  - `링크 복사` 버튼을 클릭하면 생성한 프로젝트 URL이 클립보드에 복사된다.
  - `시작하기` 버튼을 클릭하면 생성한 프로젝트 대시보드(`/projects/{slug}`)로 이동한다.
  - 이동한 대시보드가 방금 생성한 프로젝트에 해당한다. (생성한 프로젝트로 접근 가능 — Test Purpose 충족)

## Step 1 필수 입력 검증 시나리오

### Test Purpose

사용자가 Step 1 필수 항목(프로젝트 이름·이용약관 동의·만 14세 이상 확인)을 채우지 않으면 다음 단계로 진행되지 않는지 검증한다.

### Pre-condition

- 랜딩페이지가 렌더링되어 있다.
- 비인증 플로우이므로 storageState가 필요 없다. → `chromium`(미인증) 프로젝트 `testMatch`에 `**/scenario/project/**/*.spec.ts`가 포함돼야 한다.
- 단계 검증만 확인하므로 DB write가 없어 teardown이 필요 없다.

### Scenario

1. 랜딩페이지에 접근한다.
2. 안내 모달이 출력된다면 종료한다.
3. 무료로 시작하기 버튼을 클릭해 생성 모달을 연다.
4. (Step 1) 필수 항목을 비운 채 프로젝트 생성 시작 버튼을 클릭한다.
5. 필수 항목(이름·이용약관 동의·만 14세 이상 확인)을 모두 채운 뒤 다시 프로젝트 생성 시작 버튼을 클릭한다.

- Feature: 프로젝트 생성 — Step 1 필수 입력 검증
- Scenario: Step 1 필수 항목 미입력 시 다음 단계로 진행되지 않는다
- Given
  - 비인증 상태로 프로젝트 생성 모달이 Step 1에 열려 있다.
- When
  - 프로젝트 이름·이용약관 동의·만 14세 이상 확인을 비운 채 `프로젝트 생성 시작` 버튼을 클릭한다.
- Then
  - Step 2로 진행되지 않고 Step 1에 머문다.
  - 누락된 항목별 인라인 검증 에러 메시지가 노출된다. (프로젝트 이름 / 이용약관 동의 / 만 14세 이상 확인)
  - 세 항목을 모두 채운 뒤 `프로젝트 생성 시작`을 클릭하면 Step 2(식별번호)로 진행된다.

## Step 2 식별번호 불일치 검증 시나리오

### Test Purpose

Step 2에서 식별번호와 식별번호 확인이 일치하지 않으면 생성 단계로 진행되지 않는지 검증한다.

### Pre-condition

- 랜딩페이지가 렌더링되어 있다.
- 비인증 플로우이므로 storageState가 필요 없다. → `chromium`(미인증) 프로젝트 `testMatch`에 `**/scenario/project/**/*.spec.ts`가 포함돼야 한다.
- 단계 검증만 확인하므로 DB write가 없어 teardown이 필요 없다.

### Scenario

1. 랜딩페이지에 접근한다.
2. 안내 모달이 출력된다면 종료한다.
3. 무료로 시작하기 버튼을 클릭해 생성 모달을 연다.
4. (Step 1) 프로젝트 이름·이용약관 동의·만 14세 이상 확인을 입력하고 프로젝트 생성 시작 버튼을 클릭한다.
5. (Step 2) 서로 다른 값으로 식별번호/식별번호 확인을 입력하고 프로젝트 생성하기 버튼을 클릭한다.
6. 식별번호와 확인값을 동일하게 맞춘 뒤 다시 프로젝트 생성하기 버튼을 클릭한다.

- Feature: 프로젝트 생성 — Step 2 식별번호 불일치 검증
- Scenario: Step 2 식별번호와 확인값이 다르면 다음 단계로 진행되지 않는다
- Given
  - 비인증 상태로 Step 1을 통과해 Step 2(식별번호)에 도달해 있다.
- When
  - 서로 다른 값으로 식별번호와 식별번호 확인을 입력한다.
  - `프로젝트 생성하기` 버튼을 클릭한다.
- Then
  - Step 3로 진행되지 않고 Step 2에 머문다.
  - `식별번호가 일치하지 않습니다.` 에러 메시지가 노출된다.
  - 식별번호와 확인값을 동일하게 맞춘 뒤 `프로젝트 생성하기`를 클릭하면 Step 3(확인)으로 진행된다.

## 사용자 이탈 시나리오 (모달 외부 클릭)

### Test Purpose

사용자가 프로젝트 생성 도중 모달 바깥(backdrop)을 클릭해 이탈하면, 프로젝트가 생성되지 않고 모달이 닫히며 랜딩에 머무는지 검증한다.

### Pre-condition

- 랜딩페이지가 렌더링되어 있다.
- 비인증 플로우이므로 storageState가 필요 없다. → `playwright.config.ts`의 `chromium`(미인증) 프로젝트 `testMatch`에 `**/scenario/project/**/*.spec.ts`가 포함돼야 한다.
- 이탈 흐름은 DB write가 없으므로 teardown이 필요 없다.

### Scenario

1. 랜딩페이지에 접근한다.
2. 안내 모달이 출력된다면 종료한다.
3. 무료로 시작하기 버튼을 클릭해 생성 모달을 연다.
4. (Step 1) 프로젝트 제목을 입력한다.
5. 모달 바깥(backdrop) 영역을 클릭한다.

- Feature: 프로젝트 생성 — 이탈(모달 외부 클릭)
- Scenario: 생성 중 모달 바깥(backdrop)을 클릭해 이탈하면 프로젝트가 생성되지 않는다
- Given
  - 비인증 상태로 랜딩페이지가 렌더링되어 있다.
- When
  - `무료로 시작하기` 버튼을 클릭해 프로젝트 생성 모달을 연다.
  - (Step 1) 프로젝트 제목을 입력한다.
  - 모달 바깥(backdrop) 영역을 클릭한다.
- Then
  - 프로젝트 생성 다이얼로그(`role="dialog"`)가 닫힌다.
  - 랜딩페이지에 머물러 있고 어떤 프로젝트 대시보드로도 이동하지 않는다.
  - 모달을 다시 열면 Step 1부터 시작하며 이전 입력값이 비어 있다. (컴포넌트 언마운트로 상태 초기화)

### 구현 메모 (두 이탈 시나리오 공통)

- 모달은 `role="dialog"` + `aria-modal="true"` + `aria-labelledby="create-project-title"` 보유. 닫힘은 `getByRole('dialog')` 부재로 단언한다.
- 이탈 경로는 4가지: ① backdrop 클릭(`step < 4`) ② Step 1 `돌아가기`(text 버튼) ③ Step 2 우상단 `닫기`(X, `aria-label="닫기"`) ④ Step 3 `취소`. 네 경로 모두 부모 `LendingCta`의 `onClick`으로 모달을 언마운트한다.
- **ESC 핸들러가 없다** → ESC 이탈 시나리오는 작성하지 않는다(작성 시 항상 실패). suite 모달과 달리 포커스 트랩/ESC가 적용돼 있지 않다.
- 분석 이벤트: backdrop·Step 2 `닫기`·Step 3 `취소`는 `PROJECT_CREATE_EVENTS.ABANDON { step }`을 전송한다. **Step 1 `돌아가기`만 ABANDON 트래킹 없이 단순 close** — 트래킹까지 단언하려면 경로별로 분기한다.
- Step 4(성공) 화면은 이미 프로젝트가 생성된 상태라 이탈이 아니다. 이 단계에서 backdrop을 클릭해도 ABANDON 트래킹은 일어나지 않고 모달만 닫힌다.
- `LendingCta`가 `{isCreateModalOpen && <ProjectCreateForm/>}` 조건부 렌더이므로, 닫으면 언마운트되고 재진입 시 `step`·`useForm`이 초기화된다. "재진입 시 Step 1 빈 폼" 단언의 근거.
- 트리거 버튼 텍스트는 `무료로 시작하기`(`aria-label="무료로 프로젝트 생성 시작하기"`). backdrop은 dialog 박스와 형제인 overlay 요소이므로, 박스 바깥 영역을 클릭해야 닫힌다(박스 위 클릭은 닫히지 않음).
