## Test Purpose
사용자가 테스트케이스 작성을 완료하고 작성한 테스트케이스가 목록에 반영되는지 검증한다.

## Pre-condition
- `sample-project`(식별번호 `123123123`)가 시드돼 있다.
- 인증 통과 세션(`chromium-auth` / `storageState`)으로 진입한다. → 이 스펙은 인증이 필요하므로 `playwright.config.ts`의 `chromium-auth.testMatch`에 `**/scenario/cases/**/*.spec.ts`가 포함돼야 한다.

## Scenario
1. 사이드메뉴의 테스트케이스 페이지로 접근한다.
2. 우측 상단 테스트 케이스 생성 버튼을 클릭한다.
3. 테스트케이스 제목을 입력한다.
4. 테스트 스위트를 선택한다. (선택안해도 됨)
5. 테스트 유형을 선택한다. (선택안해도 됨)
6. 테스트 분류를 입력한다.
7. 사전 조건을 입력한다.
8. 테스트 단계를 입력한다.
9. 기대결과를 입력한다.
10. 테스트 케이스 생성 버튼을 입력한다.

---

- Feature: 테스트케이스 작성
- Scenario: 사용자가 테스트케이스 작성을 완료하고 작성한 테스트케이스가 목록에 반영되는지 검증한다.
- Given
  - 인증된 세션으로 프로젝트 대시보드(`/projects/{slug}`)에 진입해 있다.
- When
  - 사이드메뉴의 테스트케이스 페이지로 접근한다.
  - 우측 상단 테스트 케이스 생성 버튼을 클릭한다.
  - 테스트케이스 제목을 입력한다.
  - (선택) 테스트 스위트를 선택한다.
  - (선택) 테스트 유형을 선택한다.
  - (선택) 테스트 분류를 입력한다.
  - (선택) 사전 조건을 입력한다.
  - (선택) 테스트 단계를 입력한다.
  - (선택) 기대결과를 입력한다.
  - 테스트 케이스 생성 버튼을 클릭한다.
- Then
  - 생성 폼이 닫힌다. (`테스트 케이스 생성` 헤딩이 사라진다)
  - 테스트 케이스 목록에 입력한 제목이 노출된다.
  - 페이지를 새로고침해도 해당 제목이 목록에 남아 있다. (optimistic이 아닌 실제 영속 확인)
