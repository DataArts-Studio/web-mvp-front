# 마일스톤 관리

일정 기반으로 테스트를 계획하고 관리하는 방법을 안내합니다.

---

## 마일스톤이란?

마일스톤은 특정 기간 동안 달성해야 할 테스트 목표를 정의하는 단위입니다. 스프린트, 릴리즈, 또는 특정 이벤트에 맞춰 테스트 계획을 수립할 수 있습니다.

### 활용 예시

- **스프린트**: Sprint 1, Sprint 2, Sprint 3
- **릴리즈**: v1.0 Release, v2.0 Release
- **이벤트**: QA Sign-off, UAT, Production Deploy

---

## 마일스톤 목록

프로젝트 사이드바에서 **"마일스톤"** 메뉴를 클릭하면 전체 마일스톤 목록을 볼 수 있습니다.

### 목록 정보

| 항목 | 설명 |
|------|------|
| 이름 | 마일스톤 이름 |
| 기간 | 시작일 ~ 종료일 |
| 상태 | Planned, In Progress, Done |
| 진행률 | 완료된 테스트 비율 |

---

## 마일스톤 생성

### 빠른 생성

1. 대시보드의 **"빠른 시작"** 섹션에서 **"마일스톤 생성"** 클릭
2. 또는 마일스톤 목록에서 **"+ 새 마일스톤"** 버튼 클릭

### 입력 필드

| 필드 | 필수 | 설명 |
|------|------|------|
| 이름 | ✓ | 1~50자, 마일스톤 이름 |
| 설명 | | 최대 500자, 마일스톤 설명 |
| 시작일 | | 마일스톤 시작 날짜 |
| 종료일 | | 마일스톤 종료 날짜 |

---

## 마일스톤 상태

| 상태 | 설명 |
|------|------|
| **Planned** | 계획됨, 아직 시작 전 |
| **In Progress** | 진행 중 |
| **Done** | 완료됨 |

---

## 마일스톤 상세

마일스톤을 클릭하면 상세 화면으로 이동합니다.

### 상세 화면 구성

- **헤더**: 마일스톤 이름, 기간, 상태
- **설명**: 마일스톤에 대한 설명
- **진행률**: 프로그레스 바로 시각화
- **테스트 케이스**: 마일스톤에 포함된 케이스 목록

---

## 마일스톤에서 테스트 실행 생성

마일스톤 상세 화면에서 **"테스트 실행 생성"** 버튼을 클릭하면:

1. 새 테스트 실행이 생성됩니다
2. 마일스톤에 포함된 테스트 케이스가 자동으로 선택됩니다
3. 테스트 실행 화면으로 이동합니다

---

## 마일스톤 수정

1. 마일스톤 목록에서 수정할 마일스톤 클릭
2. 상세 화면에서 이름, 설명, 기간 수정
3. **"저장"** 버튼 클릭

---

## 마일스톤 삭제

1. 마일스톤 상세 화면에서 **"삭제"** 버튼 클릭
2. 확인 모달에서 삭제 확인

> **참고**: 마일스톤을 삭제해도 연결된 테스트 케이스는 삭제되지 않습니다.

---

## 이용 제한

베타 기간 동안 프로젝트당 최대 **10개**의 마일스톤을 생성할 수 있습니다.
