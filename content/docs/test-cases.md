# 테스트 케이스 관리

테스트 케이스를 작성하고 관리하는 방법을 안내합니다.

---

## 테스트 케이스란?

테스트 케이스는 특정 기능이나 시나리오를 검증하기 위한 테스트 단위입니다. 각 테스트 케이스는 사전 조건, 테스트 단계, 예상 결과를 포함합니다.

---

## 테스트 케이스 목록

프로젝트 사이드바에서 **"테스트 케이스"** 메뉴를 클릭하면 전체 테스트 케이스 목록을 볼 수 있습니다.

### 목록 정보

| 항목 | 설명 |
|------|------|
| 케이스 키 | 자동 생성되는 고유 식별자 |
| 이름 | 테스트 케이스 이름 |
| 상태 | Untested, Pass, Fail, Blocked |
| 태그 | 분류를 위한 태그 |
| 수정일 | 마지막 수정 일시 |

### 필터링 및 검색

- **검색**: 케이스 이름으로 검색
- **상태 필터**: 특정 상태의 케이스만 표시
- **태그 필터**: 특정 태그가 있는 케이스만 표시

---

## 테스트 케이스 생성

### 빠른 생성

1. 대시보드의 **"빠른 시작"** 섹션에서 **"테스트 케이스 생성"** 클릭
2. 또는 테스트 케이스 목록에서 **"+ 새 케이스"** 버튼 클릭

### 입력 필드

| 필드 | 필수 | 설명 |
|------|------|------|
| 이름 | ✓ | 1~200자, 테스트 케이스 제목 |
| 테스트 타입 | | 테스트 종류 (기능, 회귀, 통합 등) |
| 태그 | | 최대 10개, 분류용 태그 |
| 사전 조건 | | 테스트 실행 전 필요한 조건 |
| 테스트 단계 | | 테스트 수행 절차 |
| 예상 결과 | | 테스트 성공 시 기대되는 결과 |

---

## 테스트 케이스 상태

| 상태 | 설명 | 색상 |
|------|------|------|
| **Untested** | 아직 실행하지 않음 | 회색 |
| **Pass** | 테스트 통과 | 녹색 |
| **Fail** | 테스트 실패 | 빨간색 |
| **Blocked** | 실행 불가 (차단됨) | 주황색 |

---

## 테스트 케이스 수정

1. 케이스 목록에서 수정할 케이스를 클릭
2. 사이드 패널 또는 상세 화면에서 내용 수정
3. **"저장"** 버튼 클릭

---

## 테스트 케이스 삭제

1. 케이스 상세 화면에서 **"삭제"** 버튼 클릭
2. 확인 모달에서 삭제 확인

> **주의**: 삭제된 테스트 케이스는 복구할 수 없습니다.

---

## 테스트 케이스 보관

당장 사용하지 않지만 삭제하지 않고 보관하고 싶은 케이스는 **보관(Archive)** 기능을 사용하세요.

- 보관된 케이스는 목록에서 숨겨집니다
- 필터를 통해 보관된 케이스를 볼 수 있습니다
- 언제든지 보관 해제하여 다시 활성화할 수 있습니다
