# 테스트 스위트 관리

테스트 케이스를 그룹화하여 관리하는 방법을 안내합니다.

---

## 테스트 스위트란?

테스트 스위트는 관련된 테스트 케이스들을 묶어서 관리하는 그룹입니다. 기능별, 모듈별, 또는 테스트 유형별로 케이스를 분류할 수 있습니다.

### 활용 예시

- **기능별**: 로그인 스위트, 결제 스위트, 검색 스위트
- **유형별**: 스모크 테스트, 회귀 테스트, 성능 테스트
- **우선순위별**: Critical 스위트, High 스위트, Low 스위트

---

## 테스트 스위트 목록

프로젝트 사이드바에서 **"테스트 스위트"** 메뉴를 클릭하면 전체 스위트 목록을 볼 수 있습니다.

### 목록 정보

| 항목 | 설명 |
|------|------|
| 이름 | 스위트 이름 |
| 케이스 수 | 포함된 테스트 케이스 개수 |
| 상태 | 활성/보관 상태 |
| 수정일 | 마지막 수정 일시 |

### 필터링

- **검색**: 스위트 이름으로 검색
- **상태 필터**: 전체, 활성, 보관됨

---

## 테스트 스위트 생성

### 빠른 생성

1. 대시보드의 **"빠른 시작"** 섹션에서 **"테스트 스위트 생성"** 클릭
2. 또는 스위트 목록에서 **"+ 새 스위트"** 버튼 클릭

### 입력 필드

| 필드 | 필수 | 설명 |
|------|------|------|
| 이름 | ✓ | 10~200자, 스위트 제목 |
| 설명 | | 스위트에 대한 설명 |

> **참고**: 스위트 이름은 최소 10자 이상 입력해야 합니다. 스위트의 용도를 명확히 알 수 있는 이름을 사용하세요.

---

## 스위트에 케이스 추가

### 방법 1: 스위트 상세에서 추가

1. 스위트 상세 화면으로 이동
2. **"케이스 추가"** 버튼 클릭
3. 추가할 케이스 선택

### 방법 2: 케이스 생성 시 스위트 지정

1. 테스트 케이스 생성 시 스위트 선택
2. 케이스가 해당 스위트에 자동으로 추가됨

> **참고**: 스위트에 포함되지 않은 케이스는 [테스트 케이스 목록](/docs?tab=test-cases)에서 **"미분류"** 필터로 확인할 수 있습니다.

---

## 테스트 스위트 상세

스위트를 클릭하면 상세 화면으로 이동합니다.

### 상세 화면 구성

- **헤더**: 스위트 이름, 설명, 수정 버튼
- **케이스 목록**: 스위트에 포함된 테스트 케이스
- **통계**: 상태별 케이스 수

---

## 스위트에서 테스트 실행 생성

스위트 상세 화면에서 바로 [테스트 실행](/docs?tab=test-runs)을 생성할 수 있습니다. 스위트에 포함된 테스트 케이스가 자동으로 선택됩니다.

---

## 테스트 스위트 수정

1. 스위트 목록에서 수정할 스위트 클릭
2. 상세 화면에서 이름, 설명 수정
3. **"저장"** 버튼 클릭

---

## 테스트 스위트 삭제

1. 스위트 상세 화면에서 **"삭제"** 버튼 클릭
2. 확인 다이얼로그에서 삭제 확인

> **참고**: 스위트를 삭제해도 포함된 테스트 케이스는 삭제되지 않습니다. 케이스는 "미분류" 상태로 유지됩니다.
